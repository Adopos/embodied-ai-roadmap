# 00_总体架构

## 1. 什么是具身智能体

这里的“具身智能体”不是单个 AI 模型，也不是单纯的机器人硬件，而是一个能够与真实或仿真环境交互的完整闭环系统。

它需要具备以下能力：

```text
接收任务
→ 感知环境
→ 理解自身和世界状态
→ 调用历史经验
→ 做出决策
→ 进行规划
→ 控制身体
→ 执行动作
→ 评价结果
→ 从反馈中学习
```

一句话概括：

> 具身智能体 = 智能大脑 + 机器人身体 + 环境交互 + 反馈学习闭环。

---

## 2. 总体模块划分

一个完整的具身智能体可以拆成 12 个核心模块：

```text
1. 交互模块
2. 任务管理模块
3. 感知模块
4. 状态 / 世界模型模块
5. 记忆模块
6. 决策模块
7. 规划模块
8. 控制模块
9. 执行模块
10. 反馈 / 评价模块
11. 学习模块
12. 安全 / 约束模块
```

这些模块不是孤立存在的，而是共同组成一个循环系统。

---

## 3. 总体信息流

具身智能体的核心信息流如下：

```text
人类 / 环境 / 系统指令
        ↓
交互模块
        ↓
任务管理模块
        ↓
决策模块 ← 记忆模块
        ↓        ↑
规划模块 ← 状态 / 世界模型模块 ← 感知模块 ← 传感器
        ↓
控制模块
        ↓
执行模块
        ↓
外部环境
        ↓
反馈 / 评价模块
        ↓
学习模块
        ↺ 更新感知、状态、记忆、决策、规划、控制

横向监督：
安全 / 约束模块
```

更简洁地说：

```text
交互
→ 任务
→ 感知
→ 状态
→ 记忆
→ 决策
→ 规划
→ 控制
→ 执行
→ 反馈
→ 学习
```

## 4. 各模块的作用

| 模块          | 核心问题      | 作用                          |
| ----------- | --------- | --------------------------- |
| 交互模块        | 外部输入是什么？  | 接收语言、语音、API、界面输入，并输出结构化任务语义 |
| 任务管理模块      | 要完成什么？    | 定义目标、成功条件、失败条件、约束和任务优先级     |
| 感知模块        | 看到了什么？    | 从摄像头、LiDAR、IMU、传感器中提取环境信息   |
| 状态 / 世界模型模块 | 现在世界是什么样？ | 维护机器人位置、地图、物体状态、环境状态和自身状态   |
| 记忆模块        | 过去发生过什么？  | 保存历史任务、地图、经验、失败案例和长期知识      |
| 决策模块        | 现在该做什么？   | 根据任务、状态、记忆和约束选择当前策略或子任务     |
| 规划模块        | 具体怎么做？    | 生成任务步骤、路径、轨迹和动作序列           |
| 控制模块        | 怎么稳定执行？   | 将规划结果转化为速度、力矩、PWM、关节角等控制量   |
| 执行模块        | 身体如何行动？   | 通过电机、舵机、机械臂、底盘、无人机等执行动作     |
| 反馈 / 评价模块   | 做得好不好？    | 判断任务成功与否，统计指标，分析失败原因        |
| 学习模块        | 下次如何更好？   | 根据反馈、奖励、示范和数据更新模型或策略        |
| 安全 / 约束模块   | 哪些事情不能做？  | 横向监督决策、规划、控制和执行，保证系统安全      |

## 5. 核心关系

### 5.1 任务、状态、规划的关系

```text
任务管理模块：定义目标状态
状态 / 世界模型模块：维护当前状态
规划模块：从当前状态规划到目标状态
```

例如：

```text
当前状态：红色杯子在桌子上
目标状态：红色杯子在厨房
规划模块：导航到桌子 → 抓取杯子 → 导航到厨房 → 放下杯子
```

---

### 5.2 决策、规划、控制的关系

```text
决策模块：决定现在做什么
规划模块：决定具体怎么做
控制模块：保证动作稳定执行
```

例如无人机配送：

```text
决策：电量低，先去充电
规划：规划到充电桩的路径
控制：控制无人机沿路径飞行
执行：无人机真实移动
```

---

### 5.3 感知、状态、记忆的关系

```text
感知模块：从传感器中提取信息
状态 / 世界模型模块：把感知结果整合成当前世界状态
记忆模块：保存历史状态、地图、任务和经验
```

例如：

```text
感知：检测到前方障碍物
状态：更新局部地图中的障碍物位置
记忆：记录该区域曾经出现过障碍
决策：下次经过该区域时提前减速或绕行
```

---

### 5.4 反馈、评价、学习的关系

```text
反馈 / 评价模块：判断结果好坏
学习模块：根据结果改进系统
```

例如：

```text
反馈：抓取杯子失败
评价：失败原因是夹爪力度不足
学习：下次提高夹爪力度，或调整抓取姿态
```

## 6. 具身智能体的运行例子

任务：

```text
把桌子上的红色杯子放到厨房。
```

### 6.1 交互模块

接收用户指令：

```text
“把桌子上的红色杯子放到厨房。”
```

解析成结构化语义：

```text
任务类型：移动物体
目标物体：红色杯子
起点：桌子
终点：厨房
```

---

### 6.2 任务管理模块

定义任务：

```text
目标状态：
红色杯子位于厨房。

成功条件：
杯子稳定放置在厨房区域。

失败条件：
找不到杯子；
抓取失败；
杯子掉落；
机器人碰撞；
任务超时。

约束条件：
不能撞人；
不能打翻杯子；
电量不足时先充电。
```

---

### 6.3 感知模块

通过传感器观察环境：

```text
摄像头检测桌子和红色杯子
深度相机估计杯子距离
LiDAR 检测障碍物
IMU 和编码器提供运动信息
```

---

### 6.4 状态 / 世界模型模块

整合当前状态：

```text
机器人位置：客厅入口
红色杯子位置：桌子上
厨房位置：地图右侧
当前电量：65%
机械臂状态：空闲
障碍物：桌子旁边有人
```

---

### 6.5 记忆模块

调用历史经验：

```text
厨房入口较窄，需要减速
红色杯子容易滑，需要更稳定的抓取姿态
桌子旁边经常有障碍物，需要靠近后重新检测
```

---

### 6.6 决策模块

判断当前应执行的策略：

```text
电量充足
机械臂可用
地图已知
可以执行取杯子任务
```

决策结果：

```text
开始执行“取杯子并送到厨房”任务。
```

---

### 6.7 规划模块

生成任务步骤：

```text
1. 导航到桌子附近
2. 搜索并定位红色杯子
3. 规划机械臂抓取姿态
4. 抓取杯子
5. 导航到厨房
6. 规划放置动作
7. 放下杯子
```

同时生成：

```text
路径规划：当前位置 → 桌子附近 → 厨房
运动规划：机械臂从当前位姿 → 抓取位姿 → 放置位姿
```

---

### 6.8 控制模块

将规划结果转化为控制量：

```text
底盘线速度
底盘角速度
左右轮速度
机械臂关节角度
夹爪开合力度
轨迹跟踪误差修正
```

---

### 6.9 执行模块

真实执行动作：

```text
底盘移动到桌子旁
机械臂伸出
夹爪闭合
杯子被抓起
机器人移动到厨房
机械臂放下杯子
```

---

### 6.10 反馈 / 评价模块

检查任务结果：

```text
杯子是否在厨房？
杯子是否稳定？
有没有碰撞？
有没有掉落？
任务耗时多少？
电量消耗多少？
```

输出评价：

```text
任务成功 / 任务失败
失败原因
任务耗时
能耗
路径长度
抓取稳定性
```

---

### 6.11 学习模块

根据结果更新经验：

```text
保存成功抓取姿态
记录安全路径
更新失败案例
调整下次抓取参数
优化后续路径选择
```

---

### 6.12 安全 / 约束模块

在整个过程中横向监督：

```text
检测是否靠近人类
检测是否会碰撞
检测电量是否过低
检测速度是否过快
检测机械臂是否超过关节限制
检测通信是否中断
必要时触发急停或重新规划
```

## 7. 从 0 到 1 的构建顺序

虽然完整系统包含 12 个模块，但不能一开始全部实现。

建议按照以下顺序逐步构建：

```text
阶段 1：二维智能体
阶段 2：仿真机器人
阶段 3：真实控制系统
阶段 4：ROS 2 + SLAM + 导航
阶段 5：机器人学习
阶段 6：语言驱动具身智能体
阶段 7：VLA / 世界模型 / 通用智能体
```
